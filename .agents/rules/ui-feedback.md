# UI 피드백 및 알림 활용 규칙 (UI Feedback & Notification Guidelines)

에이전트는 사용자와의 상호작용 시 발생하는 알림 및 피드백을 구현할 때 아래의 UX 원칙을 반드시 준수해야 한다.

### 1. 토스트 메시지 (Toast Message) 활용 원칙
사용자의 작업 흐름을 방해하지 않으면서 정보를 전달해야 하는 경우, 모달창 대신 토스트 메시지(`showToast`)를 최우선으로 활용한다.

- **단순 유효성 검사 실패:** 필수 입력 항목 누락, 형식 오류 등 (예: "쿠폰 번호를 입력해 주세요.")
- **비로그인 상태의 접근:** 로그인이 필요한 기능을 클릭했을 때의 경고.
- **가벼운 상태 알림:** 복사 완료, 설정 저장 중 등.
- **API 에러 응답:** 서버 통신 중 발생한 일반적인 오류 메시지 표기.

### 2. 모달창 (Modal/Alert) 활용 원칙
사용자의 명확한 인지나 확인이 반드시 필요한 '중요한 사건'에는 모달창(`showAlert`, `showConfirm`)을 활용한다.

- **핵심 작업 성공:** 회원가입 완료, 주문/결제 성공, 쿠폰 등록 성공 등 축하와 인지가 필요한 상황.
- **파괴적인 작업 수행 전:** 데이터 삭제, 주문 취소 등 되돌리기 어려운 작업을 실행하기 전 확인 과정 (`showConfirm`).
- **중대한 시스템 오류:** 서비스 이용이 불가능할 정도의 심각한 오류 알림.

### 3. 기술적 구현 표준
- **함수 호출:** 레이아웃에 정의된 전역 함수(`showToast`, `showAlert`, `showConfirm`)를 사용한다.
- **토스트 자동 삭제:** 토스트는 노출 후 3초 뒤에 자동 삭제(`toast-exit` 애니메이션 적용)되어야 한다.
- **색상 구분:** 
  - 경고/에러: `bg-red-500` 또는 `bg-[#ec3713]`
  - 일반 정보: `bg-[#181211]` (다크 테마)
  - 성공: 상황에 따라 다크 테마 또는 브랜드 컬러 활용.

### 4. 언어 및 가독성
- 모든 알림 문구는 명확하고 친절한 **한국어**로 작성한다.
- 문장 끝에는 가급적 마침표를 사용하여 신뢰감을 준다.

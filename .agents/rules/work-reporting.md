# 작업 보고 및 기록 규칙 (Work Reporting Guidelines)

에이전트는 모든 작업(Directive)을 완료한 후, 아래의 규칙에 따라 작업 결과를 보고해야 한다.

### 1. 수정 파일 목록 제공
- 작업이 완료되면 수정하거나 새로 생성한 모든 파일의 경로를 리스트 형태로 명확히 제시한다.

### 2. 작업 내용 요약
- 각 파일별로 어떤 변경 사항이 있었는지, 혹은 전체적으로 어떤 기능이 구현/수정되었는지 핵심 내용을 요약하여 설명한다.

### 3. 언어 및 톤앤매너
- 모든 보고는 **한국어**로 진행하며, `cursorrules.md`에 정의된 페르소나와 말투를 유지한다.

### 4. 시각적 가독성
- 마크다운(Markdown) 형식을 활용하여 사용자가 한눈에 작업 범위를 파악할 수 있도록 구성한다.
